Thank you for using go-radx! This document explains how to get help with the project.

Before asking for help, please check our comprehensive documentation:

- **[API Reference](https://pkg.go.dev/github.com/codeninja55/go-radx)** - Complete package documentation
- **[Examples](https://pkg.go.dev/github.com/codeninja55/go-radx#pkg-examples)** - Code examples for common use cases
- **[Development Guide](https://codeninja55.github.io/go-radx/development/)** - Contributing and development setup

## Questions & Discussion

For general questions about using go-radx:

1. **Check existing issues** - Your question may have already been answered in [GitHub Issues](https://github.com/codeninja55/go-radx/issues)
2. **Stack Overflow** - Tag your question with `go-radx` and `fhir` for community visibility
3. **GitHub Discussions** - Start a discussion in [GitHub Discussions](https://github.com/codeninja55/go-radx/discussions) (coming soon)

### Good Questions Include

- Clear description of what you're trying to achieve
- Minimal code example demonstrating the issue
- Go version and go-radx version you're using
- Relevant FHIR resource types or specifications involved
- What you've already tried

## Bug Reports

If you've found a bug, please report it via [GitHub Issues](https://github.com/codeninja55/go-radx/issues/new/choose).

### Before Reporting

1. **Search existing issues** to avoid duplicates
2. **Test with the latest version** to ensure the bug hasn't been fixed
3. **Create a minimal reproduction** to isolate the problem

### Include in Your Report

- Go version (`go version`)
- go-radx version or commit hash
- Operating system and architecture
- Minimal code example reproducing the bug
- Expected vs actual behavior
- Any relevant logs or error messages

Use the [bug report template](https://github.com/codeninja55/go-radx/issues/new?template=bug_report.md) for consistency.

## Feature Requests

We welcome feature requests! Please submit them via [GitHub Issues](https://github.com/codeninja55/go-radx/issues/new/choose).

### Good Feature Requests Include

- **Clear use case** - What problem does this solve?
- **Context** - How would this integrate with existing features?
- **FHIR specification alignment** - Reference to relevant FHIR specs if applicable
- **Examples** - How would the API look in practice?

Use the [feature request template](https://github.com/codeninja55/go-radx/issues/new?template=feature_request.md).

## Security Issues

Please follow our [Security Policy](SECURITY.md) to report security issues responsibly.

## Community Guidelines

All interactions in this project are governed by our [Code of Conduct](CODE_OF_CONDUCT.md). We expect respectful, constructive communication from all community members.

## Response Times

This is an open-source project maintained by volunteers. Response times vary:

- **Critical security issues**: Within 48 hours
- **Bugs**: Within 1 week for triage
- **Feature requests**: Within 2 weeks for initial review
- **Questions**: Best effort, community-driven

We prioritize issues based on:
- Severity and impact
- Alignment with project goals
- Quality of the report
- Community interest (upvotes, discussion)

## Getting Faster Help

To increase your chances of getting quick help:

1. **Be specific** - Vague questions get vague answers
2. **Show your work** - Share code, not just descriptions
3. **Follow templates** - They ensure you include necessary details
4. **Be patient and respectful** - Maintainers are volunteers
5. **Help others** - Contributing to discussions builds community

## Stay Updated

- **Watch the repository** for new releases and announcements
- **Star the project** to show support and stay informed
- **Follow release notes** in [CHANGELOG.md](CHANGELOG.md)

Thank you for being part of the go-radx community!
